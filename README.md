# FSS Review Platform (FY'27)

Платформа для медпредов, региональных менеджеров (РМ) и мастер-аккаунта:
заполнение отчёта (FSS + FFE + Action Plan) → отправка РМ → комментарии/возврат/одобрение →
доступ мастеру → выгрузка Excel/презентации.

Стек — тот же, что вы уже разворачивали для SmartMed:
- **Supabase** — база данных (Postgres)
- **Railway** — бэкенд (Node.js/Express)
- **Vercel** — фронтенд (React)

---

## Шаг 1 — База данных (Supabase)

1. Зайдите в свой проект Supabase (можно новый проект, отдельный от SmartMed).
2. **SQL Editor → New query** → вставьте содержимое файла `server/schema.sql` → **Run**.
3. Это создаст все таблицы, каталог из 20 препаратов и мастер-аккаунт:
   - **email:** `admin@fss.local`
   - **пароль:** `ChangeMe123!` — **обязательно смените после первого входа** (или ещё до запуска — см. ниже).
4. Скопируйте **Connection string** (Project Settings → Database → Connection string → URI). Понадобится на шаге 2.

### Как сменить пароль мастер-аккаунта до запуска
Если хотите сразу свой пароль, а не `ChangeMe123!`:
```bash
node -e "console.log(require('bcryptjs').hashSync('ВАШ_ПАРОЛЬ', 10))"
```
Полученную строку вставьте вместо хэша в последней команде `insert into users ...` в `schema.sql` перед запуском в Supabase.

## Шаг 2 — Бэкенд (Railway)

1. Создайте новый репозиторий на GitHub, залейте туда папку `server/` (можно и весь проект целиком).
2. На **railway.app** → **New Project** → **Deploy from GitHub repo** → выберите репозиторий.
3. Если в репозитории и бэкенд, и фронтенд вместе — в настройках сервиса укажите **Root Directory** = `server`.
4. **Variables** (вкладка Variables в сервисе) — добавьте:
   - `DATABASE_URL` = строка подключения из Supabase (шаг 1.4)
   - `JWT_SECRET` = любая длинная случайная строка (например, сгенерируйте `openssl rand -hex 32`)
   - `CLIENT_URL` = временно оставьте `*`, обновите на реальный домен Vercel после шага 3
5. Railway задеплоит автоматически. В **Deploy Logs** должно быть: `FSS Review server running on port ...`.
6. Скопируйте публичный URL сервиса (что-то вроде `https://fss-review-production-xxxx.up.railway.app`).

---

## Шаг 3 — Фронтенд (Vercel)

1. На **vercel.com** → **Add New → Project** → выберите тот же GitHub-репозиторий.
2. **Root Directory** = `client`.
3. Framework Preset — Vite (определится автоматически).
4. **Environment Variables** → добавьте:
   - `VITE_API_URL` = URL Railway-бэкенда из шага 2.6 (без слэша в конце)
5. Deploy. Получите URL вида `https://fss-review.vercel.app`.

## Шаг 4 — финальная настройка CORS

Вернитесь в Railway → Variables → обновите `CLIENT_URL` на реальный URL Vercel (шаг 3.5) → передеплой.
Без этого шага при входе будет ошибка `Failed to fetch` (ровно то, что было со SmartMed).

## Как пользоваться

1. Войдите как мастер (`admin@fss.local` / ваш пароль).
2. Вкладка **«Пользователи»** → создайте РМ (региональных менеджеров).
3. Затем создайте медпредов — при выборе роли «Медпред» появится обязательное поле **«РМ, к которому прикрепить»**.
4. Передайте логины/пароли сотрудникам.
5. МП заходит → выбирает месяц → заполняет FSS (план/факт по продуктам), FFE (охват врачей/аптек), Action Plan → жмёт **«Отправить на рассмотрение РМ»**.
6. РМ видит отчёт своей команды → может оставить комментарий у любого пункта, **одобрить** или **вернуть на доработку** (с обязательным комментарием).
7. После одобрения РМ отчёт автоматически появляется у мастера, и у всех троих (МП, РМ, мастер) появляются кнопки **«Скачать Excel»** и **«Скачать презентацию»**.

## Бонус и FFE — логика, зашитая в платформу

- **FSS-бонус** (Incentive Policy FY'27, MR/Senior MR): <90% — 0; 90–99.99% — 60% ставки×достижение; 100–124.99% — 100%×достижение; ≥125% — потолок 125% от ставки. Ставка (по умолчанию 15 000 000 UZS) и курс (13 000 UZS/USD) редактируются на карточке отчёта.
- **FFE-гейт**: минимум 85% среднего скора по Doctor coverage / Core doctor coverage / Core call coverage / Doctor call coverage / Pharmacy coverage (категории A и Б) — без этого порога, согласно политике, бонус не начисляется, даже если FSS выполнен. Платформа считает и подсвечивает этот скор красным/зелёным, но **не блокирует** сам расчёт FSS-бонуса автоматически — окончательное решение по выплате остаётся за компанией (как и написано в политике).

## Локальный запуск для разработки (перед деплоем)

```bash
# бэкенд
cd server
cp .env.example .env   # впишите DATABASE_URL и JWT_SECRET
npm install
npm start

# фронтенд (в другом терминале)
cd client
echo "VITE_API_URL=http://localhost:8080" > .env
npm install
npm run dev
```

## Структура проекта

```

  schema.sql        Полная схема БД + сиды (продукты, мастер-аккаунт)
  index.js          Весь API в одном файле
client/             React-фронтенд (Vite + Tailwind)
  src/pages/         Login, MasterPanel, RmPanel, MpPanel
  src/components/    ReportView (FSS/FFE/Action Plan/комментарии), Gauge
```
